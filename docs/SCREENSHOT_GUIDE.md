# Screenshot Guide

Use this order when showing the product publicly or in partner conversations.

## 1. GO

Show the cleanest possible low-risk case first.

What it proves:

- the system does not create needless noise
- low-risk changes can still move

## 2. CONDITIONAL GO

Show a case with real introduced risk that is not fully blocked.

What it proves:

- Veridion is not binary
- it can guide remediation and approvals without overblocking

## 3. NO GO

Show the clearly unsafe change.

What it proves:

- the system can decisively block changes when introduced risk is high
- policy and next steps remain explicit

## 4. Accepted Risk

Show the suppression/exception case.

What it proves:

- exceptions do not erase truth
- accepted risk remains visible
- governance stays intact

## Notes

- Crop screenshots so the decision block and key sections are readable first.
- Prefer one screenshot per outcome rather than giant stitched collages.
- Pair each image with one sentence explaining what the system concluded and why.
